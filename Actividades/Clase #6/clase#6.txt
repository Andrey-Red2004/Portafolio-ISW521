-------- Actividad 2 ---------
Respuestas:
1- el navegador gestiona el foco y sigue el orden del DOM, como el modal esta solo a nivel de CSS entonces el navegador no lo reconoce como un entorno aislado, cuando llega al ultimo elemento del modal y se pulsa TAB el foco sigue normal hacia los elementos que quedaron de fondo en la pagina.

2- se identifican los extremos: se obtiene el primer y ultimo elemento dentro del modal
- escuchar eventos: captura el evento keydown de la tecla Tab
-Redirigir el flujo: si se detecta un intento de salida, se usa preventDefault() para frenar el navegador y mover el foco manualmente

3- Pseudocodigo:
const modal = document.querySelector('.modal');
const enfocables = modal.querySelectorAll('a, button, input, [tabindex]');
const primero = enfocables[0];
const ultimo = enfocables[enfocables.length - 1];

modal.addEventListener('keydown', (e) => {
  if (e.key !== 'Tab') return; // Solo nos interesa Tab

  if (e.shiftKey && document.activeElement === primero) {
    e.preventDefault();
    ultimo.focus(); // Retroceso al final
  } else if (!e.shiftKey && document.activeElement === ultimo) {
    e.preventDefault();
    primero.focus(); // Avance al inicio
  }
});
